[TUTORIAL]Create an Arch Linux LAMP/LEMP Server!

by RainfallMuffin » Fri Jul 18, 2014 6:56 am

I just wanted to get this out real quickly. Might help some people.

This guide is not complete. There is more to come in the future.

Chapter One: The Basics
Spoiler: show
What is LAMP or LEMP? They are software bundles for Linux that helps you make a web server.
LAMP is an acronym for Linux Apache MySQL/MariaDB PHP/Perl/Python.
LEMP is an acronym for Linux nginx MySQL/MariaDB PHP/Perl/Python.

LAMP/LEMP servers can help you host blogs, forums, downloadable files, etc.

For this tutorial I will be using MariaDB and PHP.


Chaper Two: Installing Software
Spoiler: show
First you will need to install Arch Linux. (Guide) Feel free to use Manjaro, XYZ Linux (#shamelessplug), Arch Bang, or other Arch-based distros as it is 100% compatible with this tutorial.

Now, we install some stuff.

Code: Select all
# pacman -Syu

Let's update our system first.

Code: Select all
# pacman -S apache

Install Apache for a LAMP server or

Code: Select all
# pacman -S nginx

to install nginx for a LEMP server.

Let's install more stuff!
Code: Select all
# pacman -S mysql php php-gd

This will install MariaDB, (replaced mysql in arch repos, original mysql is in AUR) PHP, and a PHP module that will help with image files.


Chapter 3: Configuration (Apache)
Spoiler: show
Now we must configure some things.

Open port 80. I assume you already know how to port forward. If not, Google is your friend.

APACHE:

Edit the apache config using:

# nano /etc/httpd/conf/httpd.conf

And uncoment the following line like so:
#LoadModule unique_id_module modules/mod_unique_id.so
LoadModule unique_id_module modules/mod_unique_id.so

And now restart apache:

# systemctl restart httpd

And then enable it:

# systemctl enable httpd


Start MariaDB and the secure installation

# systemctl start mysqld && mysql_secure_installation

You will be able to enter a root password. This is the output you should get:

Enter current password for root (enter for none):
OK, successfully used password, moving on…

Setting the root password ensures that nobody can log into the MariaDB
root user without the proper authorization.

Set root password? [Y/n] y
New password:
Re-enter new password:
Password updated successfully!
Reloading privilege tables..
… Success!

By default, a MariaDB installation has an anonymous user, allowing anyone
to log into MariaDB without having to have a user account created for
them. This is intended only for testing, and to make the installation
go a bit smoother. You should remove them before moving into a
production environment.

Remove anonymous users? [Y/n] y
… Success!

Normally, root should only be allowed to connect from ‘localhost’. This
ensures that someone cannot guess at the root password from the network.

Disallow root login remotely? [Y/n] y
… Success!

By default, MariaDB comes with a database named ‘test’ that anyone can
access. This is also intended only for testing, and should be removed
before moving into a production environment.

Remove test database and access to it? [Y/n] y
– Dropping test database…
… Success!
– Removing privileges on test database…
… Success!

Reloading the privilege tables will ensure that all changes made so far
will take effect immediately.

Reload privilege tables now? [Y/n] y
… Success!

Cleaning up…

All done! If you’ve completed all of the above steps, your MySQL
installation should now be secure.

Thanks for using MariaDB!

Now open up your apache config again and paste this in

# Use for PHP 5.x:
LoadModule php5_module modules/libphp5.so
AddHandler php5-script php
Include conf/extra/php5_module.conf

Now open up /etc/php/php.ini using nano and uncomment these lines:

extension=gd.so
extension=mysql.so
extension=pdo_mysql.so

Now you have configured your web server. To see if it html works, paste this into /srv/http/index.html

<html>
<title>It Works!</title>
<body>
<h2>The server is working</h2>
</body>
</html>

And then go to your IP address in your web browser. If it says 'It Works!' your web server is working.

Now paste this in /srv/http/info.php
<?php
phpinfo();
?>

And then go to <YOURIPADDRESS>/info.php If you see a bunch of confusing words and your PHP version number, your php is working.

You are done! Now in Chapter 4 we are going to install WordPress and in Chapter 5 phpBB
