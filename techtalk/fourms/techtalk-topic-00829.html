Some weird linux program game thing I made in C++

by maxell » Mon Sep 08, 2014 4:39 am

I am going to turn this into something interesting but in the meantime here's the "engine". The Zs are walls, the player is an X, and the playfield consists of # and @. You can't do anything yet except move around, and I'm not revealing the source code yet because it's uglier than xterm's. (Maybe that's an understatement.)

It's not a big source code and the only headers I have that are not mine are <iostream> and some obscure blogger's header named "BufferToggle". I don't even need <string> since I can just use a char array for my application. So obviously the program's source isn't BIG it's just really ugly. We've got hardcoded objects, arrays never being used up to the full limit, and general code unelegance. I think the application could use much less memory if I didn't do these things and I'm going to work on optimizing it tommorow. As of now, here's a really crappy program that I call "KGame" due to it's unnecessary anti-optimization. (I was coding at like 3 AM, okay?)

I learned that I can't make an array with a size defined at runtime. I tried vectors but for some reason I got a segfault. I'll have to work on that.

Anyways the good stuff is what I'll keep so remember that the next release of this program (whatever it's called) will act pretty much the same but it will take up way less memory.
