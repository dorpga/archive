I'll try to address your issues to the best of my knowledge::

I'm taking a stab in the dark here, but I think list indices start at 0 in Python because ranges start at 0 by default. It makes shuffling through a list easier, and allows a step before reaching 1 in a for loop (using whole steps).

Python is somewhat based on C and other pre-existing standards. It's a standard convention for a return value of 0 to mean the function executed properly, and anything else should be an error number so that it can be handled appropriately, unless of course it the return value is a solution to an equation, in which case it would be handled differently (and this is where you do not want a function to return None if 0 is the answer to the equation):
for example if you have:
"def addnum(x, y): return x + y"
if any number pairs that would equal 0 are passed to the function the return result would be None, most likely causing a TypeError in the parent function and crashing your program.

What do you mean by putting brackets on if conditions?
"if True: print(True)" works perfectly fine in the latest release of Python 3, and I believe print is now a function because of the advancements made to it.
Spoiler: show

(END)


Pygame can be found here, https://bitbucket.org/pygame/pygame/downloads, which includes a Python 3.3 MSI, so in theory it should be able to be built for Python 3.4

Constants are nice when they reference an instance of an object. (:

Lemme start you off. Python has a whole bunch of functions that are like BASIC commands.

So this:

Code: Select all
PRINT "TYPE HELLO" ;Print statement
INPUT X$ ;Input statement
IF X$ == "HELLO" ;If statement
PRINT "HELLO TO YOU!" ;What happens if
END IF ;Ending the if statement


Would be more like:

Code: Select all
print("Type hello") #Send "hello" to the print function
x = raw_input() #Set x to the result of the raw_input function
if x == "hello": #If statement. Everything in the if statement is indented by 4 spaces.
    print("Hello to you!")


Keep in mind that everything after ; in QBasic and # in Python is a comment, I.e. not executed by the program.
