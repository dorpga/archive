Since OSFirstTimer is all about operating systems, it should have it's own. FirstTimerOS is a MikeOS-based operating system, so it uses x86 assembly language. It will be community-based and the plan is to have most of the code submitted by you. At the moment none of the kernel code is changed apart from "mikeos" being changed to firsttimeros, but it will improve (I'll probably learn how to do assembly some time). It is completely open-source, too.

How to compile
If you know a bit of assembly or BASIC or just want to experiment, you may want to try and rebuild the operating system. This usually only takes one second since assembly is such a low level operating system. First, download NASM. On Linux, use your package management system to install the "nasm" package. If on OS X, copy the "nasm" file from this and place in /usr/bin. If on Windows, get the installer here. To compile, just extract the source and run the script for your OS. If you are using linux/mac/bsd, you need to run it as root.

Remember, if your code improves the OS, please submit it here and I'll add it to the next version.

How to flash the OS onto a hard drive (or virtual hd/flash drive/floppy disk)
This will make it act as a real OS that you can read/write files to it. There is already a virtualbox image if you want this, but if you want to change it or do a full install onto a flash/hard drive, then read.
Virtualbox instructions
First, download a small lightweight linux distro. I recommend gparted live since it lets you double check if a FAT partition has been made. If you decide to use "Damn Small Linux" (yes that's a thing), download it from a mirror since the main site is extremely slow.

Next, move the firsttimeros.flp file somewhere else like your desktop, and rename it firsttimeros.iso. Now, add this and the OS of your choice as new disk drives under the IDE controller of virtualbox. Boot into the OS, open a terminal and type "sudo dd if=/dev/cdrom of=/dev/sda". It should wait a little bit and say something like "1.8mb copied". The OS is now on the drive and so you can remove the disks and boot to the hard drive.

Real physical hardware instructions
You would probably want to do this on a flash drive, but it works on anything, really. You can use the dd command in linux if you know what you're doing, but I recommend you download a utility like gdiskdump (linux) or Win32 Disk Imager (windows). Use one of those to flash it and you now have a bootable flash/hard drive.

What you can do to improve it (you should!)
If you know assembly or know how to make useful basic programs, then submit your code here! All code is welcome if it improves the OS and makes it unique.
