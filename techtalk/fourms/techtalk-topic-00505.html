[NEWMETA] The Linux desktop explained.

by maxell » Thu Jun 26, 2014 3:33 am

Okay, you're done with Unity, and you want to have XFCE - but you want those Kwin effects from KDE - but the right click menu from Openbox - Oh, and that GTK theme you saw on Gnome-Look, and those custom title bars...yeah, this isn't going to be as easy as clicking install is it?

Okay, so the first thing most Linux desktops need to have (at least for now) is the X window system. Running X and some usual programs won't do you any good, because X is only the base framework for splitting programs into windows. To be able to get those sick wobbly windows (or just move windows around for that matter) requires something known as...

A window manager! Unlike X, there are a whole bunch of window managers with no real dominant one. There are many, some which are extremely minimal and some which are extremely feature packed. Window managers like Compiz and Kwin have all these effects, ones like Metacity and Openbox are in the middle, and ones like xmonad and i3 are either completely awesome if you are a geek, or pathetic if you're a UX designer.

Don't see those cool borders yet? Some compositing window managers need a program to decorate the windows before it turns them into OpenGL 3D thingies (sorry, not a graphics programmer). Compiz users usually use Emerald or Metacity to decorate their windows, while Kwin users mostly opt for Aurorae and Dekorator.

Great! Now how do we launch programs? We need a program launcher. These are pretty straightforward. You could use a terminal as your program launcher, or use a panel like xfce4-panel or tint2, or even a right-click menu like MyGtkMenu or the one that comes with Openbox.

You now have a complete desktop environment, however you may be missing something if you are using, say, a barbarian distro with no package manager *cough*slackware*cough*, you'll obviously need these things:

Drivers for your graphics card. What good is a graphical OS if it can only output in 640x480 SVGA? You'll need to install drivers for your graphics card. Nvidia makes this super hard for you (if you're not using Ubuntu, who is the poor soul that cleans up Nvidia's crap) because their goal is to make your entire life a misery; however it almost seems as if AMD shares this goal.

A graphical toolkit! A toolkit like GTK or Qt allows programmers to not re-invent the wheel every five seconds. It draws the buttons, sliders, and other widgets so you - err, the programmers, don't have to. They're theme-able too!
