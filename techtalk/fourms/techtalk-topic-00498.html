Should Linux be only developed by free foundations?

by rp4k » Tue Jun 24, 2014 8:47 pm

Should consumer/server Linux be only developed by free foundations? I started wondering this after I started realizing that Ubuntu was developed by Canonical, RedHat (yes, Redhat is a company that PUBLICLY TRADES. If you don't believe me search up "RHT" on Google or Bing) was developed by Redhat, and Solaris was developed by Sun, purchased by Oracle in 2010.

The problem in a corporation developing a distro is that a company's goal is to profit, which dilutes the free part of Linux.

My view on this question is that corporations should continue developing Linux distros, because the distro being used is at the consumer's discretion.

Linux isn't developed by those companies. Yes, they contribute source code to the linux kernel, but they aren't the people backing linux as a kernel. They're just making complete OSes that use the linux kernel. Making it against their terms to make money off of linux like that would be a violation of linux's free software license, GNU GPL. (so they'd have to switch to a different license, pissing off free software enthusiasts)
