Part 1, The basics on how it works
An operating system, is the base program in a computer, but at the same time, a really complex piece of software.
The OS directs all the computers software and hardware, and makes the hardware able to share data, with the computer.

A modern OS is built up of these parts:

# 1 The Kernel - the kernel is the central part of the Operating System. Its task is to control resources used by the system (CPU usage, RAM, and Harddrive space usage), to the different programs on the on the computer.

# 2 Drivers - Drivers make hardware able to "talk" with the Operating System, making software able to for example: Read data from the CD drive, and so forth

# 3 The Shell - the shell is a piece of software which allows the user, to communicate, with the kernel, In most cases its a text based program. Where the user types program calls on a command line.

# 4 The Boot Loader - A boot loader, is the first piece of executed code that loads/boots, the OS. The code is executed by the BIOS (Basic Input/Output System), which on start checks all your drives for bootable media. It was named after this sentence: "Starting a computer without it, is as hard as dragging yourself up in your bootstraps".

Heres how it works:


The Boot Loader is loaded by the BIOS, which then loads the OS.

An basic explanation on how the input gets send back and forth:

# 1 When the OS is loaded, it receives input from the user.
# 2 The input gets send to the current application.
# 3 Which then gets sent to the kernel, which then gets all the stuff needed to execute that command.
# 4 The command then goes to the hardware, which returns the result of the command back, to the kernel.
# 5 Which then tells the application the result, which then usually tells/shows the user it.

Part 2 - Differences in kernels:
Spoiler: show
Part 2, Differences in kernels

--NOTICE: Will add pictures later. --

So, now you know the basics of how an OS works, let's get into the in-depth stuff.

The different type of kernels.

ATM there exists 3 types of kernels (That i know of):
- Monolithic Kernels
- Micro Kernels
- Hybrid/Modular Kernels.

In this episode, I'm going to teach you the differences, between the different kernel structures.

Monolithic Kernels:

A Monolithic kernel, has the software running along with the main kernel thread, and also using the same memory area. This way makes accessing hardware more powerful.

Pros:
+ Less software involved = faster speed
+ because its one piece of software, it should be smaller.
+ Less code, fewer bugs. And less security problems.

Cons:
- if a driver crashes on a monolithic kernel, it might crash the entire system, and huge kernels gets hard to maintain.
- Coding in the kernel is harder, because it cannot use common libraries.
- Bugs in code, has large side effects, including damaging some data structure, of a totally unrelated part of the kernel.
- Kernels usually tends to get large quickly, which makes them hard to maintain.
- Hard to port to other CPU structures
- Harder to code software, as developers has to implement there own routines, which makes the software buggier.

Monolithic kernels consists of core functions of the OS and the drivers. With the ability to load modules in runtime. which provides rich power over the hardware.

Micro Kernels:

Micro Kernels is an other approach, to making kernels.
Instead of running software beside the kernel as modules, it separates them out, and runs them in servers.
Those servers communicates with a mini-kernel. leaving as little as possible in the system space, and more in the user space (i will explain what those are in next part).
Only stuff witch REALLY requires being in privileged mode, are in kernel space (ICP -- Inter-Process Communication), witch is a basic version of the kernel functions.

Pros:
+ Maintenance is generally easier.
+ Patches are easier to test.
+ Quick development time, and less reboots on software debugging.
+ More persistence, if something breaks, its easier to fix.

Cons:
- More memory usage.
- More software interfacing = more performance loss.
- Messaging bugs can be harder to fix.
- Extremely Context based.
- Software will be more complex.

Hybrid Kernels:

Hybrid kernels, used in most commercial OSes.
They are similar to micro-kernels, except they have some extra code, to increase performance.
Basicly it's like if a modular and a micro kernel had a baby. xD
phillips1012 defines it as: "A buzzword created by Microsoft to describe the monolithic kernel in the NT series of microsoft windows"


Pros:
+ Faster Development time.
+ Faster build time.
+ Faster integration.

Cons:
- More interfaces = possible more bugs.
- Maintaining can be confusing.


And that's the end of this part, hope you have learned something useful.

Part 2.5, Quick Explaining.

Here's some extra small information.



User Space and Userland:
Userspace is the name of code running outside the kernel, Userland is the name of the libraries and various programs that the OS interacts with the kernel with.

Kernel Space:
Kernel space is reserved for running privileged kernel functions, and drivers.
Where, User Space is memory area where software and some drivers run.

ICP (Inter process communication):
ICP is, some methods, making the exchange of data between multiple threads, or in multiple processes possible.

This was just some quick explaining, if you want more in-depth, explanations. Then wait for later parts, or look it up on Wikipedia
