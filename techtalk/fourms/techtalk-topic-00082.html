FAQ:
Spoiler: show
What is Linux?

Linux is a kernel made by Linus Torvalds in 1991 that uses GNU tools to make it a full operating system.

What is GNU/Linux?

Read the above question first. The Linux kernel uses GNU tools to make it a full operating system. That's why some people call it GNU/Linux.

Can I get all the software that I use on Windows/Mac on Linux?

The answer is normally, but not always, no. You can get alternatives, though: you could use LibreOffice instead of MS Office, GIMP instead of Photoshop and so on. You could also use a program like Wine, that ports some Windows programs to Linux.

Is Linux easy to use?

Sometimes. Some Linux distributions are actually designed to be easy to use. Others, like Arch Linux, are designed for Linux geeks.

Should I switch to Linux?

If you want to, you can. There's a lot of guides out there in the internet on how to install Linux.

What is a Linux distribution?

A Linux distro is simply the Linux kernel with compiled packages, like the X window system.

What are some of the more common distros?

Ubuntu and Linux Mint are the most common.

How can I install Linux?

You can find a guide somewhere on the internet, but oh well, I'll post a simple one here.

1. Download UUI
2. Download the distro ISO file
3. Launch UUI
4. Follow the steps in UUI. (WARNING: This will wipe ALL your data on the selected drive. BE CAREFUL.)
5. Restart your computer
6. Stick the USB in.
7. Start your computer.

How much does Linux cost?

Nothing! It's totally free! For example, if you wanted to try Linux Mint, you would go to Mint's website and download the ISO file and use something like UUI to install it.

Can I try Linux without leaving Windows/Mac OS X?

Of course! You can either install it in a virtual machine, or dual-boot it. There's lots tutorials for both somewhere on the internet.

Can Linux only run on a PC?

No! Linux runs on mobile phones, servers, supercomputers, embedded systems like ATM's and more!



--Common Linux misconceptions--


Linux can't run any good software

Yes, it CAN. For example, do you want to create a document? Use OpenOffice or LibreOffice. Want to browse the web? Use Firefox or Chromium.

Linux is GNU based

No, Linux uses GNU software. Linux is not based off of anything. Linux is Unix-LIKE.

No one uses Linux

Actually, a LOT of people use Linux. 1 or 2% PC users, most touch-screen phones use Android, most servers use Linux, even supercomputers!


Ways to get Linux on a USB (credit to ProjectFlux)
Spoiler: show
UNETBOOTIN
This is PROBABLY the most recommended method to get a Live USB.

1. Pick a distro
(OPTIONAL) If your using ubuntu, there's a option below the ISO option.
2. When your done picking your own distro, you can advance.
( Move your data to somewhere in your computer. )
It will download the ISO for you.
4. When it's done, boot the USB via BIOS.

ISO:
If Unetbootin can't download the ISO, do it manually.
Then put it in the ISO option.

UUI
This usually works. WINDOWS ONLY!
1. Prepare your ISO's, if you have any.
2. When you selected your Distro, be sure to have the ISO laying around the folder.
3. Pick your USB letter.
4. When you advance, there will be a warning about formatting. BACK UP YOUR DATA, FIRSTTIMERS!
5. When done, test it in a VM.
If successful, it's ready for booting up.

BURNING
1. Get a CD/DVD (A DVD RW is recommended)
2. Get an ISO
3. Burn the ISO to the CD/DVD
This will get you a live CD/DVD.

VANILLA
This method is NOT suggested, but if Unetbootin, UUI and the burning method aren't working, well, i guess you can try this.
1. Extract the ISO into your USB.
2. Try it out.
If it works, well, lucky you!

I would just like to point out that UUI and Unetbootin are bug ridden. On Windows use win32diskimager instead, much more success, and yes it works with iso files, even though it defaults to .img files (floppy I think).

Linux users should use DD when they are burning an iso:

Code: Select all
[email protected]~$ sudo -i
[email protected]~# cd /where/the/iso/file/is
[email protected]~# ls /dev/sd? (Identify which one is your USB stick. Generally sdb or sdc).
[email protected]~# dd if=isofile.iso of=/usb/device bs=4M

Then reboot and boot into the environment. Yes you can use this to burn Windows and Mac iso files as well.
