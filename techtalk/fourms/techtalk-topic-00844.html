Tutorial: How to install Arch on a UEFI based system

by KomputerKid » Sat Sep 13, 2014 6:36 pm

Alright I thought I would teach you all how to setup Arch Linux on a UEFI based system. First of all some questions you might have:


Why would I want to use arch on a UEFI based system?
Well for one thing, Arch boots up significantly faster on a UEFI system then a BIOS system. When I was originally running in BIOS mode, it would be about 10 seconds. However on UEFI I'm booting up in like 3 seconds, even on a rotational hard disk.
Why would I want to use UEFI in general?
UEFI allows for more partitons on a disk, and much larger disks to be booted. I believe it maxes out at 2.9 EB (Exabytes) or 2.9 Billion Terabytes. Also the GPT Table allows for more partitions on a disk (I haven't found the max). On a BIOS MBR based system, you are limited to 2 TB disks, and four partitions. You can't go beyond 2 TB on MBR, don't even think about it, and adding more than 4 partitions takes a lot of hacking. (But people have done it) You CAN use disks larger than 2TB on a BIOS system if they are using the GPT partition table, and they aren't acting as boot disks. (As in they only are read once the OS is loaded).

Okay now for the tutorial: Installing arch on a UEFI based system.

To begin, place the CD or USB Stick containing arch into your PC and reboot.
Press the key you would normally use to get into UEFI. (F10, F12, F2, ESC, DEL)
Turn OFF SECURE BOOT if your UEFI has it.
Exit the BIOS saving the changes, reboot then re-enter the BIOS
Select the Boot tab, and choose to boot from your CD-drive or USB stick.
You should be greeted by a boot loader with options similar to this:

  • Boot Arch Linux UEFI Mode x86_64
  • UEFI Shell x86_64
  • UEFI Shell x86
  • Reboot

Select boot Arch Linux UEFI mode.

Now once you are booted into arch, run cfdisk.

This tutorial assumes that you are going to single-boot arch on a UEFI Non-Mac PC. If you would like to dual-boot with Windows, or are on a Mac read the Wiki.

1st Delete all partitions on the disk.

Next create a 100 MB Partition on /dev/sda1
on the 100 MB partition, select FS Type, and choose EFI System partition.
Now create the remaining partitions you would normally create. I generally add:

  • 100 MB /boot partition
  • 4 GB swap Partition
  • 50-75 GB / partition
  • The rest for home.

Write the changes to the disk. Partition any other disks you wish to use if you have multiple disks.
Now the tricky part:

with /dev/sda1 which we specified as the EFI partition, we need to create a fat32 file system:

Code: Select all
mkfs.vfat -F32 /dev/sda1


Then make the rest of the file systems for your other partitions. I generally do:

  • /boot ext2
  • swap for swap (duh)
  • / and /home ext4

Now MOUNT YOUR ROOT PARTITION FIRST! If you mount the other partitions first, then mount root, your fstab will become all screwed up.
Code: Select all
mount /dev/sdxx /mnt

mount /boot next
Code: Select all
mkdir /mnt/boot

Code: Select all
mount /dev/sdxx /mnt/boot

now mount your EFI partition:
Code: Select all
mkdir /mnt/boot/efi

Code: Select all
mount /dev/sdxx /mnt/boot/efi


sdxx is the partitions you created.

Continue with the install as you normally would mounting any other partitions and installing the base system. Generate your fstab, and then configure your system. We will skip ahead to when we install grub as we need some extra things:
first install dosfstools and efibootmgr
Code: Select all
pacman -S dosfstools efibootmgr


Now install grub:
Code: Select all
pacman -S grub


Install grub to your boot disk (generally /dev/sda)
Code: Select all
grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=grub --recheck --debug


Then simply run

Code: Select all
grub-mkconfig -o /boot/grub/grub.cfg


Exit the chroot, reboot, and enjoy your new arch system.
