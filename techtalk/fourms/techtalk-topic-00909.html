Mac OS X, Linux and BSD users - Shellshock bug

by allstarsnorks2 » Fri Sep 26, 2014 1:46 pm

Mac OS X, Linux and BSD users, there is a new vulnerability found in the GNU Bash shell that allows for an attacker's code to be executed once the shell is invoked. This Shellshock vulnerability, according to sources, causes more devastating effects than Heartbleed. GNU Bash shell versions vulnerable to Shellshock are version 1.14 through 4.3. Users running a version that is unlisted are safe.

Apple has yet to acknowledge this Shellshock vulnerability, so don't expect the Mac side to be safe just yet. Companies have actually released patches for their distributions. These include: Ubuntu 10.04 LTS, 12.04 LTS, 14.04 LTS, Red Hat Enterprise 4 to 7, Fedora, CentOS 5 to 7 and Debian. So if you're running this distributions, you are safe if you install the patch.

To know whether your *nix-based bug is vulnerable, copy and paste this code:
Spoiler: show
env x='() { :;}; echo vulnerable' bash -c "echo this is a test"


If vulnerable, this will be shown:
Spoiler: show
vulnerable
this is a test


If errors from bash appear, it's a sign that you are safe from this Shellshock vulnerability.
