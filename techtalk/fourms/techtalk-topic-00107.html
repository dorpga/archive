Why PHP is a horrible language

by phillips1012 » Sun May 25, 2014 5:01 am

PHP is terrible. Here are the reasons I personally hate php:

  • Weak typing: PHP doesn't give a fuck about types. The == operator is pretty much a "similar" operator, not an equality operator.
  • PHP is extremely hard to debug. Fatal errors do not return a stack trace and you cannot catch them at runtime. Many things that should throw errors fail silently.
  • One of the good things about php is that the concatenation operator is seperate from the addition operator. However, someone thought using a period "." for concatenation was a great idea. Why is this bad? You have to use "->" to access functions from instances of classes. Wonderful design right there.
  • PHP tries to stay extremely backwards compatible and has hundreds of built-in functions, such as "mysql_real_escape_string". It's pretty much the equivelant of hoarding in programming language design.
  • It's extremely insecure and the design of it doesn't help developers write secure applications at all.
  • It sucks at being object oriented. Most functions to operate on objects are in the form of built-in functions that are tied to the global namespace
  • http://www.php.net/manual/en/language.types.array.php... "This type is optimized for several different uses; it can be treated as an array, list (vector), hash table (an implementation of a map), dictionary, collection, stack, queue, and probably more." They're saying this as if it's a good thing...
  • Syntax. Did you know php, a curly-braced language, has an `endif;` statement? http://www.php.net/manual/en/control-st ... syntax.php
  • We should not be mixing logic and static files in 2014. `.php` files shouldn't be served, they should serve.

There are many more reasons php pisses me off, but these are the ones that piss me off the most.
