Webilder Error 403 Fix. 0.7.2 and below

by Leo7Mario » Mon Jul 07, 2014 1:45 am

Warning! - This post only applies to Webilder versions 0.7.2 and older...

Hi guys! First post in awhile.

If you have watched the OSFT Pinguy OS video then you will know there was a program called Webilder.
If you don't know what it is, it is a program that downloads random flickr photos tagged as "interesting" and sets them as you desktop background.
I liked it a lot and loved having it on Debian-based distros.
Then in May, flickr changed the API so that it will only take https requests.
The problem is that the flickr API Webilder uses sends http requests instead of https requests.
If you try to download flickr photos it will give you a 403 Forbidden error.
I was kind of sad that this happened. And that was why I was trying to find a fix. And... I did!

Follow these steps to fix Webilder:

1: Quit Webilder (This is important!)
2: Open up a terminal and type in "sudo su" (If it doesn't ask for a password, skip to step 4!)
3: Enter your password. If you are using a shared computer than you must have administrative privileges.
4: Now, type in cd /usr/share/pyshared/webilder/flickr
5: Make sure you have a text editor to edit the .py file. (I recommend Leafpad)
6: Now type in [text editor name ex. leafpad] flickrapi.py
7: Search the file for "http", then replace "http" with "https"
8: Make sure to save afterwards!
9: type "exit" in the terminal. After that, close the terminal
10: Open Webilder
11: Try Downloading photos from Webilder. If it works go to Step 12. If it doesn't, go back to step 1 and try again.
12: Webilder is now working again! Congrats!

It is a pretty easy fix if you have some knowledge with the Linux terminal.
Now that you applied the SSL layer to the code, you can use Webilder again! Yay!
Hope this helps.

P.S. Please excuse any grammar errors (if any). I want to try to help people that still use Webilder.
