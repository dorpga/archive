Re: Visual Basic.net

by Ham62 » Mon May 26, 2014 1:32 am

You could probably make a VLC style player in VB.NET. I personally haven't used VB.NET before but I am using Visual Basic 6.0 to create a XMPlay style music player minus the fancy custom window styles. For this player I am using the BASS library which is what XMPlay utilities and is very good. I also know that you can create a video player in VB. I guess to create a VLC style player, after you finish creating the user interface, when the user tries to load a file into the program the program would then read the file extension and based on what that is it would load the appropriate libraries to play whatever the specified file is. You would probably want to work on this one part at a time, first getting the video, or sound depending on what you want then starting on the other after the first is functional.

In short I guess you could, but because of the complexity of this project and how many different file types that VLC supports, this might take a while to complete, as would be expected in any language you attempt this in.

I hope this is what you were asking and that my answer was helpful.
